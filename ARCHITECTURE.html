🤖 The agent loop — observe → diagnose → act → verify

Lives in ai/agent.py :: run_loop(). Repeats up to retry_budget (3) times, then escalates. This same loop powers canary decisions, self-healing, and the background watcher.

1 · Observe
Read signals
signals.snapshot() — health, error-rate trend, bad pod reason. Read-only.
2 · Diagnose
Pick ONE action
decide() asks Gemini for an allowlisted action; rule_based() if the model is down.
3 · Act
Pull a safe lever
clamp() bounds the numbers, needs_human() gates it, then actuator.execute().
4 · Verify
Did it work?
verify() re-checks health. Healthy → done. Not → loop again. Out of budget → escalate.
Every decision is logged by telemetry.record_decision() Autonomy L0/L1 stops at step 3 and hands to a human

🛡️ The IaC safety gauntlet — why LLM-generated infra is safe

Generated Terraform is guilty until proven valid. It must clear all six gates, in order, before terraform apply. The tools are the judge — the model can't talk its way past them.

GateToolCatchesCode
1 · Formatterraform fmtGarbage / malformed outputCI
2 · Validateterraform validateMade-up args, unknown resourcesCI
3 · Planterraform planAnything that wouldn't buildCI
4 · Policyconftest / OPAPublic DB, bad region, too many nodes, missing tagsinfra/policy/policy.rego
5 · Contract3-line checkMissing required outputscheck_contract()
6 · Destroy-guardread the planWould destroy/replace your databaseassert_nondestructive()

Before gate 1, two pure-Python pre-checks already run at generation: check_allowlist() (only approved resource types) and check_contract(). Fail → RCA → regenerate, up to max_fix_attempts, then escalate.

🔌 "Plug into a socket, not a wall"

One idea runs through everything: program to a contract, so each moving part is swappable without touching the rest.

Socket #1 · Cloud

platform.json → Terraform

Every cloud must deliver 3 outputs: kubeconfig, database_url, app_endpoint. Add a cloud = one JSON block. The LLM generates that cloud's Terraform on demand.

Socket #2 · Orchestrator

Actuator interface

The agent only ever calls actuator.execute(action, params). Today it drives Kubernetes/Helm. Swap in a new Actuator subclass and nothing upstream changes.

Socket #3 · AI

LLMClient + Actuator

A reasoning socket (ask() → text) and an acting socket. Swap Gemini for any model by writing one class. Thinking and doing are each swappable.

🔒 The allowlist — the cage the AI cannot open

guardrails.py. The model can only ever return a name from this list — every one is reversible. Dangerous verbs simply aren't expressible.
